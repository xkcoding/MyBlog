# Design: Refine Design System

## Current State Analysis

### 现有配色方案

```css
/* Light Mode */
--background: #fdfdfd;  /* 近白色背景 */
--foreground: #282728;  /* 深灰前景 */
--accent: #006cac;      /* 蓝色强调 */
--muted: #e6e6e6;       /* 浅灰静音色 */
--border: #ece9e9;      /* 淡边框 */

/* Dark Mode */
--background: #212737;  /* 深蓝灰背景 */
--foreground: #eaedf3;  /* 浅色前景 */
--accent: #ff6b01;      /* 橙色强调 */
--muted: #343f60;       /* 深蓝静音色 */
--border: #ab4b08;      /* 橙棕边框 - 过于强烈 */
```

### 现有 Logo 分析

- **概念**：代码日记本 (日记本 + `</>` 代码符号)
- **结构**：矩形日记本 + 装订线 + 装订孔 + 代码符号 + 底部横线
- **颜色**：使用 CSS 变量 `--accent` 动态适配主题

## Design Decisions

### DD-1: 配色优化策略

**决策**：在现有配色基础上进行微调，保持风格一致性

**理由**：
- 用户明确表示保持现有风格
- 现有配色已建立视觉识别
- 仅需解决对比度和协调性问题

**具体调整**：

| 变量 | Light Mode | Dark Mode | 调整说明 |
|------|------------|-----------|----------|
| `--background` | **重新设计** | 保持 | 亮色背景需优化 |
| `--foreground` | **重新设计** | 保持 | 配合新背景 |
| `--accent` | **重新设计** | 保持/微调 | 亮色主题色需重新选择 |
| `--muted` | **重新设计** | 保持 | 配合整体配色 |
| `--border` | **重新设计** | 保持/微调 | 配合整体配色 |

**设计方向**：
- 亮色模式：完全重新设计，使用**紫色系**作为强调色（用户偏好）
- 暗色模式：基本保持现有配色（#212737 背景 + #ff6b01 橙色强调色的组合效果良好）

**亮色模式配色思路**：
- `--accent`：紫色系（如 #7c3aed、#8b5cf6、#6d28d9 等）
- 整体配色围绕紫色强调色构建和谐的色彩体系

### DD-2: Logo 重设计方向

**决策**：保持"代码日记本"核心概念，优化视觉细节

**设计原则**：
1. **简洁性**：减少不必要的装饰元素
2. **可识别性**：在小尺寸下保持清晰
3. **一致性**：与整体设计语言协调
4. **适应性**：支持亮色/暗色模式

**优化方向**：
- 优化日记本形状比例
- 简化装订线设计
- 增强代码符号的视觉重量
- 移除底部横线（降低复杂度）

### DD-3: 色彩对比度标准

**决策**：遵循 WCAG 2.1 AA 标准

| 使用场景 | 最低对比度 |
|---------|-----------|
| 正文文本 | 4.5:1 |
| 大号文本 (≥18pt) | 3:1 |
| UI 组件边界 | 3:1 |

### CSS 变量结构（保持不变）

```css
@theme inline {
  --color-background: var(--background);
  --color-foreground: var(--foreground);
  --color-accent: var(--accent);
  --color-muted: var(--muted);
  --color-border: var(--border);
}
```

## Alternatives Considered

### Alt-1: 完全重新设计配色

**否决原因**：用户明确要求保持现有风格

### Alt-2: 使用设计 Token 系统

**否决原因**：当前规模不需要额外复杂度，现有 CSS 变量方案已足够

### Alt-3: Logo 使用渐变效果

**否决原因**：增加复杂度，不利于小尺寸显示和打印
